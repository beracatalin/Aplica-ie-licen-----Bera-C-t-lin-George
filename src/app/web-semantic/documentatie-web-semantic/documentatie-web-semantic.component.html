WEB SEMANTIC

     Principiile pe care se bazează sintaxele Microdata și RDFa sunt:

  1. orice porțiune a conținutului paginii poate fi marcată cu semnificația sa (de exemplu un șir de cifre poate fi marcat cu declarația că e vorba de un număr de telefon al cuiva);
  2. porțiunea marcată devine valoarea unei proprietăți pentru un anume subiect (entitatea despre care e vorba în pagină); subiectul e marcat la rândul său acordându-i un identificator Web (URI) iar dacă nu facem asta subiectul devine nod anonim, responsabilitatea identificării sale rămâne în sarcina clientului care va culege datele (de exemplu motorul de căutare);
  3. orice referință inclusă în pagină (atributele SRC și HREF) poate de asemenea să devină valoarea unei proprietăți;
  4. la nevoie, se pot adăuga și proprietăți invizibile unui cititor uman, folosind tag-uri HTML care nu afectează conținutul vizibil al paginii (SPAN-uri sau DIV-uri goale, META, LINK).

     Se observă că RDFa folosește anumite atribute HTML dedicate:

  1. PREFIX pentru declararea prefixelor;
  2. ABOUT pentru indicarea subiectului propozițiilor (entitatea despre care e vorba în interiorul unui tag HTML);
  3. TYPEOF pentru declararea tipului subiectului;
  4. PROPERTY pentru atribute - proprietățile ale căror valori sunt de regulă fragmente din conținutul paginii;
  5. REL pentru relații - proprietățile ale căror valori sunt alte URI, preluate de la alte atribute ABOUT conținute;

    Microdata este o versiune mai primitivă și mai simplă decât RDFa – folosește aceleași principii de marcare a informației, însă atributele dedicate sunt altele:

  1. ITEMID este subiectul (similar cu ABOUT);
  2. ITEMTYPE este tipul subiectului (similar cu TYPEOF)
  3. ITEMPROP este proprietate (similar cu PROPERTY și REL, fără a se face diferențiere);
  4. ITEMSCOPE este un atribut boolean (valoarea sa poate fi vidă) – simpla sa prezență anunță faptul că în tag-ul curent intenționăm să încorporăm un graf despre ceva.

    În MySQL, SELECT se folosește pentru a selecta unul sau mai multe elemente din tabel; UPDATE pentru a actualiza unul sau mai multe elemente din tabel, iar DELETE pentru a șterge elemente din tabel.
